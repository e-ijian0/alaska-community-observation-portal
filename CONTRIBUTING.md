# Contributing Guidelines

Thank you for your interest in contributing to the Alaska Community Observation Portal! This project aims to support rural Alaskan communities in sharing observations on hydrological changes that affect subsistence safety and cultural practices.

## How to Contribute

### Reporting Issues
- Use GitHub Issues to report bugs, suggest new features, or discuss improvements.
- When reporting observations or data issues, please include details about location, date, and environmental conditions.

### Submitting Data
- Data submissions should follow the JSON schema defined in `data_schema.json`.
- Ensure observations include accurate location coordinates (latitude/longitude) or place names.
- Include relevant notes about safety conditions or other contextual information.

### Code Contributions
- Fork the repository and create a new branch for your changes.
- Follow the existing code style and include appropriate documentation.
- Submit a Pull Request (PR) with a clear description of the changes.
- Ensure your PR passes any existing tests (if applicable).

### Community Guidelines
- Respect local knowledge and cultural practices.
- Collaborate with community members to ensure tools are appropriate and useful.
- Prioritize safety and accuracy in all observations.

### Questions?
Feel free to open an issue for any questions about contributing or using the portal.
